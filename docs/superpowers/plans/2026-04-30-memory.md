## RUTAS API EXISTENTES

**Frontend `/api/`:**
- `POST /api/chat` — Chat en capítulos (Haiku)
- `POST /api/symptoms` — Registrar vitales + extracción de síntomas (Haiku)
- `GET /api/aliis/insight` — Insight diario (Haiku, 1/día cacheado)
- `GET /api/aliis/notify` — Cron 9am: genera notificaciones
- `POST /api/aliis/push/subscribe` — Registrar token Web Push
- `POST /api/stripe/checkout` — Crear sesión checkout
- `POST /api/stripe/subscribe` — Crear suscripción directa con trial 14 días
- `POST /api/stripe/coupon` — Validar promotion code de Stripe
- `POST /api/stripe/setup-intent` — Setup intent para pago
- `POST /api/stripe/portal` — Portal de gestión de suscripción
- `POST /api/stripe/webhook` — Webhook de eventos Stripe
- `GET /api/notifications` — Listar notificaciones del usuario
- `PUT /api/notifications/[id]` — Marcar como leída
- `DELETE /api/notifications/[id]` — Eliminar notificación
- `POST /api/notifications/read-all` — Marcar todas como leídas
- `POST /api/invite/validate` — Validar código de invitación
- `POST /api/invite/consume` — Consumir código
- `POST /api/account/delete` — Eliminar cuenta

**Backend Express `api.aliis.app`:**
- `POST /pack/generate` — Generación de pack completo (Sonnet 4.6)
- Webhook Stripe (en routes/stripe.ts)

## DECISIONES DE ARQUITECTURA (NO CAMBIAR SIN DISCUTIR)

1. **Backend gana en rate limits**: Si frontend y backend difieren, el backend es la verdad. El backend aplica "1 pack cada 7 días" para Free.

2. **Migrations solo aditivas**: Nunca `DROP TABLE`, nunca `DROP COLUMN`. Siempre `ADD COLUMN IF NOT EXISTS`.

3. **Feature flags en Supabase**: No en código ni env vars. La tabla `feature_flags` es la fuente de verdad.

4. **Prompt caching preservado**: Cualquier migración de modelos debe mantener el caching en system prompts de Anthropic.

5. **Enricher es responsabilidad del backend**: El frontend no construye contexto médico — lo hace `backend/src/lib/enricher.ts`.

6. **Pack generation nunca falla silenciosamente**: Si hay error en generator, devolver error explícito al usuario.

7. **RLS en todas las tablas nuevas**: Sin excepción. Toda tabla nueva tiene `ENABLE ROW LEVEL SECURITY` y al menos una política.

8. **No shared package hasta S5-A**: Antes del mobile, `types.ts` se copia manualmente si se necesita en ambos lados.
